int类型

  1. int类型的分类

    • TINYINT

      • 存储空间：1字节

      • 取值范围：有符号 [-128, 127]，无符号 [0, 255]

    • SMALLINT

      • 存储空间：2字节

      • 取值范围：有符号[-32768, 32767]，无符号[0, 65535]

    • MEDIUMINT

      • 存储空间：3字节

      • 取值范围：有符号[-8388608, 8388607]，无符号 [0, 16777215]

    • INT

      • 存储空间：4字节

      • 取值范围：有符号[-2147483648, 2147483647]，无符号[0, 4294967295]

    • BIGINT

      • 存储空间：8字节

      • 取值范围：有符号[-9223372036854775808, 9223372036854775807]，无符号[0, 18446744073709551615]

  2. int类型的使用

    • 自增长ID

      推荐使用bigint

    • unsigned or signed

      据实际情况使用，一般情况下推荐默认的sigend

      unsigned 的注意事项

  3. int (N)

    • int(N) 和 zerofill

      • int(N)中的N是显示宽度，不表示存储的数字的长度的上限。

      • zerofill表示当存储的数字长度< N时，用数字0填充左边，直至补满长度N

      • 当存储数字的长度超过N时，按照实际存储的数字显示

      int(N)中的 N 和 zerofill 配合才有意义，且仅仅是显示的时候才有意义，和实际存储没有关系，不会去截取数字的长度。

  4. auto_increment

    • 自增

    • 每张表一个

    • 必须是索引的一部分

    innodb_autoinc_lock_mode：{0|1|2}: 分别表示的是传统，连续，交错。mysql5.7中默认是1，有AI锁

    • innodb auto increment计数器初始化

      • 如果为表指定auto_increment列，则innodb数据字典表中包含一个为自动递增计数器的特殊计数器，用于为该列分配新增，该计数器仅存在内存中，而不存在于磁盘上

      • 要在服务器重启启动后初始化自动递增计数器，innodb在第一次插入包含列的表时执行与以下语句等同

      select max(ai_col) from table_name for update;

      • 重启mysql，如果存储引擎是MyISAM，自增主键的最大ID不会丢失，因为MyISAM的自增主键的值保存到数据文件中，而innodb是保存到内存中

      • 当auto_increment整数列用完时，后续insert操作返回重复键错误

      • 当你重启mysql服务器时，innodb可能会重复使用为列生成但从未存储过的旧值auto_increment(即，在回滚的旧事务期间生成的值)

      • auto_increment的值不一定连续，因为如果回滚了，在没有重启的情况下，下次插入就会跳过回滚的值

      相关参数：

      auto_increment_increment：递增步长

      auto_increment_offset：起始值

数字类型

  1. 数字类型的分类

    • 单精度类型：float

      • 占用空间：4字节

      • 精确性：低

    • 双精度类型：double

      • 占用空间：8字节

      • 精确性：低，比float高

    • 高精度类型：ducimal

      • 占用空间：变长

      • 精确性：非常高

  2. N和字符集

    • char(N)

      假设当前table的字符集的 最大长度 为 W , 则 char(N) 的最大存储空间为 (N X W)Byte ;假设使用 UTF-8 ，则char(10)可以最小存储10个字节的字符，最大存储30个字节的字符，其实是另一种意义上的 varchar ◦ 当存储的字符数小于N时，尾部使用空格填充，并且填充最小字节的空格

      • test_char 表实际二进制存储文件， 可以使用 shell> hexdump -C test_char.idb验证

  3. blob和text

    • 在blob和text上创建索引时，必须制定索引的长度

    • blob和text列不能有默认值

    • blob和text列排序只能使用该列的前max_sort_length个字节

json

  1. json介绍

    • json是一种轻量级的数据交换语言，并且是独立于语言的文本格式。

    • 一些nosql数据库选择json作为其数据存储格式

    • mysql 5.7.x开始支持json数据类型

  2. json vs blob

    • json

      • json数据可以做有效性检查

      • json使得查询性能提升

      • json支持部分属性索引，通过虚拟列的功能可以对json中的部分数据进行索引

    • blob

      • blob类型无法中数据库层做约束性检查

      • blob进行查询，需要遍历所有字符串

      • blob只能做指定长度的索引

      5.7之前，只能把json当作blob进行存储，数据库层面无法对json数据做一些操作，只能由应用程序处理。

  3. 结构化和非结构化

    • 结构化

      • 二维表结构(行和列)

      • 使用sql语句进行操作

    • 非结构化

      • 使用key-value格式定义数据，无结构定义

      • value可以嵌套key-value格式的数据

      • 使用json进行实现
