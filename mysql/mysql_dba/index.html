mac mysql install

  1. 安装

    brew install mysql@5.7

  2. 设置环境变量

    ln -s /opt/homebrew/opt/mysql@5.7/ /opt/mysql/mysql

    echo PATH=/opt/mysql/mysql/bin:$PATH

  3. 设置root账户密码

    mysql_secure_installation

  4. 登录验证

    mysql -uroot -ppassword

    1. 配置mycli登录

      db_local_mysql = mysql://root:happy998518@localhost:3306

    2. 登录

      mycli -dsn db_local_mysql

    说明：使用brew安装mysql，如果是mac m1的话，目前只能安装mysql5.7。默认的data目录中/opt/homebrew/var/mysql。查看my.cnf的文件位置，可以使用mysqld --help -vv | grep -10 my.cnf。默认中/opt/homebrew/etc/my.cnf

mysql 5.7版本以前都是有一个默认的数据库test，所有用户默认都有权限的，5.7以后基于安全考虑去掉了test数据库

mysql的版本升级

  • 安全关闭mysql服务

    mysqld stop

  • 设置软链，将mysql的软链接由升级前的版本指向升级后的版本

    ln -s ..

    这时其实是可以启动mysql服务的，但是观察mysql的日志，由很多警告

  • 使用mysql_upgrade -s -p升级

    回退的话，升级之前备份data下的mysql目录，系统表

mysql的参数

  • 查看当前参数的值

    select @@autocommit; 查看当前回话

    select @@global.autocommit; 查看全局

    如果参数既是session参数，又是global参数的话，修改global的，只对以后建立的数据库链接有效，已经存在会话是不修改，修改的参数不保存到配置文件中，需要修改配置文件

    但是修改 check_proxy_users 的全局参数的值，当前会话生效。check_proxy_users是全局参数。

grant all privileges on . to 'root'@'127.0.0.1' identified by 'admin123' with grant option;

with grant option; 设置这个用户可以给别的用户grant

删除权限 revoke 只删除权限，不删除用户，也就是用户还可以登录。

  • 查看警告

    show warnings;

  • 查看某个用户拥有哪些权限

    show grants for 'root'@'localhost'

mysql的角色

使用的proxy来模拟角色(mysql 5.7才支持的)

前提：开启启用代理用户映射

  1. set global check_proxy_users = 1;

  2. set global mysql_native_password_proxy_users = 1;

  1. 创建proxy用户 和 需要代理的用户

    create user 'shopcrm_dba'@'127.0.0.1' identified by 'admin123';

    create user 'shopcrm_001'@'127.0.0.1' identified by 'admin123';

    create user 'shopcrm_002'@'127.0.0.1' identified by 'admin123';

  2. 将代理用户赋予权限

    grant select, insert on . to 'shopcrm_dba'@'127.0.0.1';

  3. 将需要的代理的用户赋予代理用户的权限

    grant proxy on 'shopcrn_dba'@'127.0.0.1' to 'shopcrm_001'@'127.0.0.1';

    grant proxy on 'shopcrn_dba'@'127.0.0.1' to 'shopcrm_002'@'127.0.0.1';

    如果登录的用户没有proxy。需要执行下面的sql，赋予代理权限

    grant proxy on ''@'' to 'root'@'127.0.0.1' with grant option;

  4. 使用shopcrm_001登录

慢sql日志

记录的是大于设置慢sql查询的阈值的，没有等于

  • 相关参数

    slow_query_log
    是否开启慢查询日志

    slow_query_log_file
    ◦ 慢查询日志文件名, 在 my.cnf 我们已经定义为slow.log，默认是 机器名-slow.log

    long_query_time
    制定慢查询阈值, 单位是秒，且当版本 >=5.5.X ，支持毫秒。例如 0.5 即为 500ms
    大于该值，不包括值本身。例如该值为2，则执行时间正好等于2的SQL语句不会记录

    log_queries_not_using_indexes
    将没有使用索引的SQL记录到慢查询日志
    如果一开始因为数据少，查表快，耗时的SQL语句没被记录，当数据量大时，该SQL可能会执行很长时间 需要测试阶段就要发现问题，减小上线后出现问题的概率

    log_throttle_queries_not_using_indexes
    限制每分钟内，在慢查询日志中，去记录没有使用索引的SQL语句的次数;版本需要 >=5.6.X
    因为没有使用索引的SQL可能会短时间重复执行，为了避免日志快速增大，限制每分钟的记录次数

    min_examined_row_limit
    扫描记录少于改值的SQL不记录到慢查询日志
    结合去记录没有使用索引的SQL语句的例􏰀，有可能存在某一个表，数据量维持在百行左右，且没有建立索引。这种表即使不建立索引，查询也很快，扫描记录很小，如果确定有这种表，则可以通过此参数设置，将这个SQL不记录到慢查询日志。

    log_slow_admin_statements
    记录超时的管理操作SQL到慢查询日志，比如ALTER/ANALYZE TABLE

    log_output
    慢查询日志的格式，[FILE | TABLE | NONE]，默认是FILE;版本 >=5.5
    如果设置为TABLE，则记录的到 mysql.slow_log

    log_slow_slave_statements
    在从服务器上开启慢查询日志 • log_timestamps
    写入时区信息。可􏰁据需求记录UTC时间或者服务器本地系统时间

如果慢sql文件比较大，可以tail -10000 ｜ mysqldumpslow

  • 将慢sql日志保存到表中

    表 myslq.slow_log

    set global log_output = 4; // 1是NONE，2是FILE，4是TABLE

    show variables like 'log_output%'

    如果想用table到话，需要将slow_log表到存储引擎有默认到csv格式修改成MyISAM。备份数据的时候，需要忽略mysql.slow_log表 _

    show engines;

    show create table slow_log;

    alter table slow_log engine = MyISAM;

数据类型

  • 整数类型

    1. unsigned or signed

◦ 据实际情况使用，一般情况下推荐默认的sigend

◦ unsigned 的注意事项

  2. int(N) 和 zerofill

◦ int(N)中的N是显示宽度，不表示存储的数字的长度的上限。
◦ zerofill表示当存储的数字长度< N时，用数字0填充左边，直至补满长度N

◦ 当存储数字的长度超过N时，按照实际存储的数字显示

  • 字符串类型

    1. char(N)和varchar(N)

      n表示的最大字符，很字符集有关，不是最大字节数。

    2. 可以使用exdump -C test_char.idb 验证char和varchar的区别

  • 日期类型

    timestamp带时区

    1. now()和sysdate()的区别

      select now(), sysdate(), sleep(5), now(), sysdate();

      或者

      select now(6), sysdate(6);
