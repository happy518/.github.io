表整理

大表，指单个数据文件磁盘占用大于100G，或者单个表数据记录量大于1亿。可以根据自己具体环境确定。

查看表占空间和记录数

常规表数据清理

常规表指没达到大表标准的。

  • Delete

    语法：Delete from table_name [ where condition]

    Delete 只删除符合条件的数据，不会减少表所占空间。
    Delete大量数据后，会存在碎片，需要整理回收碎片空间
    optimize table table.name 或者 alter table table.name engine=‘innodb’ (会锁表，注意在业务低谷期执行)

  • Truncate

    语法：Truncate table table_name

    Truncate 删除全表数据，回收所占表空间。

  • Drop

    语法：Drop table table_name

    Drop 删除全表数据和表结构，回收所占表空间。

大表删除

大表 Drop 表的时候，所有Mysql的相关进程都会停止，直到Drop结束，耗时时间很长，若是生产环境，会阻塞业务访问。出现这情况是因为，在drop table的时候，innodb维护了一个全局锁，drop完毕锁才释放。

mysql的drop机制

  • 冲缓冲池中删除数据，这个时候会维护一个table cache的全局锁，导致任何与表相关的操作都无法执行，直到drop完毕，锁才会释放

表空间管理方式

  • 共享表空间

    某一个数据库的所有表数据，索引文件都放在一个文件夹中，默认这个共享表空间的文件路径在datadir目录下，默认文件名ibdata1

  • 独立表空间

    每个表都将生成以独立文件方式来进行存储，每个表都有一个.frm文件(表结构定义文件)，和一个.ibd文件。frm文件保存了美国表的元数据，包括表结构定义，该文件与数据库存储引擎无关，ibd文件保存了每个表的数据和索引

  • linux硬链接

    linux操作系统上存在的文件都会有一个node index指向它。

    此外还有一个文件名指向这个node Index。

    硬链接就是不止一个文件名指向node Index，同时存在几个文件名指向node Index。（类似桌面快捷方式）
